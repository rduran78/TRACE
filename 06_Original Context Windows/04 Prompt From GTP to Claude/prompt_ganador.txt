### Option A: Raster or grid focal operations

If the cells form a regular grid or can be represented as a raster, consider using `terra::focal()` or an equivalent method to compute moving-window statistics.

For example:

```r
neighbor_mean <- terra::focal(r, w = matrix(1, 3, 3), fun = mean, na.rm = TRUE)
neighbor_max  <- terra::focal(r, w = matrix(1, 3, 3), fun = max,  na.rm = TRUE)
neighbor_min  <- terra::focal(r, w = matrix(1, 3, 3), fun = min,  na.rm = TRUE)
```

Adapt this only if it is methodologically equivalent to the current neighbor definition.

### Option B: Precomputed adjacency table with `data.table`

If raster focal operations are not appropriate, consider building a long neighbor table once:

```text
cell_id | neighbor_cell_id
```

Then join values by year and aggregate using keyed `data.table` operations:

```text
cell_id | year | mean_neighbor_x | max_neighbor_x | min_neighbor_x
```

### Option C: Alternative optimization

If, after reading the script, you find a better approach, use it. You are not required to follow Option A or Option B if another strategy is more appropriate, faster, safer, or more faithful to the current methodology.
